# ⚡ 快速開始指南

## 🎯 一分鐘上手

### 前端開發（這個視窗）

```bash
# 1. 修改前端代碼後，執行：
./前端提交.sh "你的更改描述"

# 例如：
./前端提交.sh "優化購物車 UI"
```

### 後台開發（另一個視窗）

```bash
# 1. 修改後台代碼後，執行：
./後台提交.sh "你的更改描述"

# 例如：
./後台提交.sh "新增訂單篩選功能"
```

### 檢查狀態（隨時可用）

```bash
./檢查狀態.sh
```

---

## 📋 文件分工

### 前端文件（這個視窗負責）
- `index.html`, `products.html`, `cart.html` 等前端頁面
- `js/main.js`, `js/front-products.js` 等前端 JS
- `css/` 目錄下的所有樣式文件

### 後台文件（另一個視窗負責）
- `admin-*.html` 所有後台頁面
- `js/admin-*.js` 後台 JavaScript
- `backend/` 整個後端目錄

---

## ⚠️ 重要提醒

1. **提交前自動同步**：腳本會自動執行 `git pull`，確保代碼最新
2. **避免衝突**：不同視窗編輯不同文件，避免同時修改同一文件
3. **明確訊息**：提交訊息要清楚描述更改內容
