# 快速開始開發

## 解決本地開發緩存問題

### 🚀 方法 1：使用開發服務器腳本（最推薦）

在終端機執行：
```bash
./dev-server.sh
```

這個腳本會：
- 自動檢查並安裝 http-server（如果需要的話）
- 在端口 8080 啟動服務器
- **完全禁用緩存**（修改後即時更新）
- 自動打開瀏覽器

### 🔧 方法 2：手動啟動 http-server

```bash
# 安裝 http-server（如果還沒安裝）
npm install -g http-server

# 啟動服務器（禁用緩存）
http-server -p 8080 -c-1 --cors -o
```

參數說明：
- `-p 8080`: 使用端口 8080
- `-c-1`: 禁用緩存（永不緩存）
- `--cors`: 啟用跨域請求
- `-o`: 自動打開瀏覽器

### 💻 方法 3：使用瀏覽器開發者工具

1. 打開瀏覽器開發者工具（F12 或 Cmd+Option+I）
2. 點擊右上角設定圖標
3. 在 **Network** 區域勾選：
   - ✅ **Disable cache**（停用緩存）
4. **重要**：開發者工具必須保持開啟，這個設定才有效

### 🔄 方法 4：每次硬刷新

修改文件後使用硬刷新：
- **Mac**: `Cmd + Shift + R`
- **Windows**: `Ctrl + Shift + R` 或 `Ctrl + F5`

### 👻 方法 5：無痕模式

使用無痕模式可以避免大部分緩存：
- **Chrome/Edge**: `Cmd + Shift + N` (Mac) 或 `Ctrl + Shift + N` (Windows)
- **Firefox**: `Cmd + Shift + P` (Mac) 或 `Ctrl + Shift + P` (Windows)

## 推薦工作流程

### 最佳組合（最快效率）：

1. **使用開發服務器腳本**：
   ```bash
   ./dev-server.sh
   ```

2. **開啟瀏覽器開發者工具**，勾選 **Disable cache**

3. 完成！現在修改任何文件都會即時看到更新，無需推送到 GitHub！

## 驗證緩存是否已禁用

1. 打開瀏覽器開發者工具（F12）
2. 進入 **Network** 標籤
3. 刷新頁面
4. 檢查請求的 **Status Code**：
   - ✅ 應該是 `200` 或 `200 (from disk cache)`（首次）
   - ❌ 不應該是 `304 (Not Modified)`

## 常見問題

### Q: 為什麼還是看不到更新？
A: 檢查：
1. 是否使用了正確的服務器（http-server，不是 python http.server）
2. 是否開啟了瀏覽器開發者工具的「Disable cache」
3. 是否在正確的端口（8080）訪問
4. 文件是否真的保存了（檢查文件修改時間）

### Q: 修改 CSS 後看不到變化？
A: 確保：
1. CSS 文件路徑正確
2. 瀏覽器沒有緩存 CSS（檢查 Network 標籤）
3. 選擇器是否正確（檢查 Elements 標籤）

### Q: 服務器啟動失敗？
A: 檢查：
1. 端口 8080 是否被其他程序佔用
2. 是否安裝了 Node.js 和 npm
3. 權限是否正確（可能需要 sudo）

## 停止服務器

按 `Ctrl + C` 停止服務器

---

**提示**：使用 `./dev-server.sh` 是最簡單的方式，一次設定，永久有效！
